硬件综合评分系统算法说明文档 (V2.0)

1. 设计目标
   本算法旨在通过软件识别的硬件型号和基本参数，模拟出接近 UserBenchmark、Geekbench 或 天梯图 的相对性能评分。系统采用百分制（0-100），并引入“木桶效应”来评估整机的平衡性。
2. 核心组件评分规则
   2.1 CPU (处理器) 评分
   CPU 评分不再单纯依赖主频，而是采用 “基础分 + 代际补偿 + 频率修正” 的模型。
   维度 权重/逻辑 说明
   基础分 (Base) 型号档次 (Tier) i9/R9: 85分; i7/R7: 70分; i5/R5: 55分; i3/R3: 40分
   代际补偿 (Gen) 架构演进 Intel 12代+/AMD 5000+: +15分; 8-11代: 0分; 7代及以前: -15分
   特例加成 苹果 M 系列 M1/M2/M3 系列由于 IPC 极高，给予 +40 分的起步补偿
   频率修正 动态主频 以 3500MHz 为基准，每增加 200MHz 加 1 分，上限 ±10 分
   公式： Score = Clamp(Base + GenBonus + FreqOffset)
   2.2 GPU (显卡) 评分
   显卡评分以 “架构效能 (80%) + 显存规格 (20%)” 为核心，并对移动端设备进行修正。
   架构识别：通过预设的型号列表（RTX 4090=100, RTX 3060=72 等）确定基准分数。
   显存阶梯：
   24GB (生产力级): 100分
   12GB - 16GB (现代主流): 85-90分
   8GB (及格线): 70分
   < 4GB (入门级): 25-40分
   移动端惩罚：非 RTX 40 系列的笔记本显卡（Laptop/Mobile），由于功耗限制，评分自动乘以 0.85。
   2.3 RAM (内存) 评分
   内存评分重点在于 “容量 (核心) + 代际带宽 (辅助)”。
   容量 (GB) 基础分 评价
   64GB+ 100 生产力/顶级
   32GB 92 优秀
   16GB 75 现代标准
   8GB 50 入门办公
   < 8GB 25 瓶颈
   频率加成：DDR5 (4800MHz+) 或更高频率增加 5分。
   老旧扣分：DDR3 或更低规格扣除 10分。
   2.4 Disk (硬盘) 评分
   硬盘是系统响应速度的决定因素，分值跨度巨大。
   NVMe (PCIe 4.0+): 100分 (识别关键词: Gen4, 980 Pro, SN850等)。
   NVMe (标准型): 92分。
   SATA SSD: 75分。
   HDD (机械硬盘): 30分 (在现代操作系统中属于严重性能拖累)。
3. 场景化评分算法 (应用表现)
   为了反映真实的使用体验，场景评分采用加权计算：
   3.1 💻 编程开发 (Coding Score)
   重点关注编译速度和多任务并行能力。
   公式：CPU _ 45% + RAM _ 40% + Disk _ 15%
   说明：显卡性能对纯编程几乎无影响。
   3.2 🤖 AI/大模型 (LLM Score)
   极度依赖显存容量和显卡架构。
   公式：VRAM_Score _ 70% + GPU_Model _ 20% + RAM _ 10%
   说明：显存是“入场券”，低于 8GB 会产生剧烈分值衰减。
   3.3 🚀 综合总分 (Overall Score)
   引入 “木桶效应” 指数，避免单项极高但单项极低导致的体验偏差。
   公式：(加权平均分 _ 90%) + (最低项得分 _ 10%)
   加权比重：CPU(35%) + GPU(30%) + RAM(20%) + Disk(15%)
4. 等级划分标准 (Grading)
   根据总分，将设备划分为五个等级，对标市场定位：
   等级 分数区间 市场定义
   S 90 - 100 旗舰级：顶级生产力工具，可胜任 4K 剪辑/深度学习。
   A 75 - 89 高端：主流游戏与专业办公的高效机型。
   B 60 - 74 主流：流畅运行 Win11，满足日常多任务处理。
   C 45 - 59 入门：能够使用，但在大型软件下有明显延迟。
   D < 45 过时/瓶颈：建议升级硬件，无法满足现代流畅度要求。
5. 算法演进方向 (未来建议)
   多核权重：目前算法对多核(Multi-core)能力的识别较弱，未来应引入 Cores/Threads 参数。
   功耗墙识别：针对轻薄本，引入 TDP 限制因子，修正其在高负载下的表现。
   显卡架构补丁：定期更新 GPU 列表，将 Tensor Core 和 RT Core 的版本纳入考量。

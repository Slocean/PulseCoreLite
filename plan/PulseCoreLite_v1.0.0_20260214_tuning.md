PulseCoreLite 全面调优计划（体积、资源、性能）

一、目标与基线
1. 体积目标
   - 前端产物压缩体积降低 20%~40%
   - Tauri 可执行文件体积降低 10%~25%
2. 资源目标
   - 空闲状态 CPU 降低 30% 以上
   - 内存占用降低 10%~20%
3. 性能目标
   - UI 刷新稳定，卡顿与丢帧明显减少
   - 采样周期抖动下降

二、体积优化
1. 字体与图标减量
   - 仅保留实际使用的 Inter 字重与 JetBrains Mono 字重
   - Material Symbols 替换为 SVG 子集或最小化字体包
2. 生产构建产物收敛
   - 关闭 production sourcemap
   - 移除未使用静态资源
3. Tauri/Rust 产物收敛
   - 开启 LTO
   - 开启 strip
   - 设置 panic=abort
   - 降低 codegen-units

三、资源占用优化
1. 采样分层
   - 高频：CPU/内存/网络（1s）
   - 低频：磁盘/温度/GPU 频率（2s~5s）
2. 窗口不可见降频
   - 隐藏/最小化时将刷新间隔提升到 3s~5s
3. GPU 采样减负
   - PowerShell 采样间隔放大或可配置开关

四、性能提升
1. 前端渲染分层
   - 高频数据与低频数据拆分 store
   - 仅更新开启展示的模块
2. 样式渲染降负
   - 高频区域减少重阴影与复杂渐变
   - 进度条与数值显示尽量避免重绘成本高的效果
3. 采样循环稳定化
   - 使用 interval 驱动，减少漂移

五、落地优先级
1. 高收益低风险
   - 字体与图标减量
   - 关闭 sourcemap
   - Rust release profile 收敛
2. 中收益中改动
   - 采样分层与降频
   - 窗口不可见降频
   - GPU 采样间隔提升
3. 深度优化
   - 前端渲染分层
   - sysinfo 刷新策略细化

六、验收方式
1. 体积
   - dist 目录与最终安装包大小对比
2. 资源
   - 空闲与高负载场景 CPU/内存对比
3. 性能
   - UI 刷新稳定性与丢帧观察
